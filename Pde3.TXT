📘 Guia Técnico para Desenvolvedores e Equipe
Para garantir que qualquer pessoa que assuma o projeto entenda o fluxo de trabalho, utilize estas definições:

1. Camada de Visualização (src/pages/)
Esta pasta contém as entidades de página. No React Router, cada arquivo aqui corresponde a uma experiência de usuário distinta. Se você precisa alterar a lógica de como um protocolo é filtrado, o lugar é aqui.

2. Componentização (src/components/)
Aqui residem os elementos que mantêm a consistência visual. A navbar.jsx é o melhor exemplo: ela é escrita uma única vez e importada nas páginas necessárias, garantindo que o menu seja idêntico em todo o sistema.

3. Arquitetura de Estilo (src/styles/)
Adotamos o CSS Modular por Página. Isso significa que o estilo da tela de detalhes.jsx não interfere no login.jsx. O arquivo global.css deve ser usado exclusivamente para definir a paleta de cores oficial (Azul #007bff) e fontes.

4. Lógica de Roteamento (src/app.jsx)
Este é o arquivo de controle. Ele utiliza a lógica de location.pathname para decidir, por exemplo, que a Navbar não deve aparecer na tela de Login, mas deve estar presente em todas as outras. É aqui que as rotas dinâmicas (como /acompanhar/:id) são declaradas.

5. Ponto de Entrada Estático (index.html)
Diferente de sites comuns, no React este arquivo raramente é alterado, exceto para configurar o SEO, o título da página e o ícone da aba (Favicon).
